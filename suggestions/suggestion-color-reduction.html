Color Reduction + Mobile Card Layout

Reduce from 6 competing hues to a strict 3-color + PnL system. Reorganize mobile card view into 2-column pairs to cut card height in half.

Part 1 — Color Palette

Daily Yield → softer green · Downside Risk → white · Decay Rate → light red

Current — 6 Hues

Violet — accent
Green — positive, exponent badges
Amber — points, decay, boosts
Red — danger, downside risk
Cyan — gradient endpoint
Teal — modal gradient

Proposed — 3 + PnL

Violet — sole accent
White — data, points, risk, primary
Muted — labels, N/A, zeroes
PnL Signal Only:
Soft green — Daily Yield positive
Soft red — Decay Rate
Amber — removed
Saturated red — removed
Saturated green — removed
Cyan — removed
Teal — removed

Color Rules Going Forward

RULE 1
Only violet for accents
Active states, badges, boosts, gradients, focus rings — all violet. Zero amber, teal, cyan.
RULE 2
Color = signal only
Soft green (#86EFAC) for positive yield. Soft red (#FCA5A5) for decay. White for everything else.
RULE 3
Pastel PnL tones
Even signal colors are desaturated — no #10B981 or #EF4444. Gentler on dark backgrounds.
RULE 4
Hierarchy = brightness
White → text → text-2 → muted. Stand out via brightness, not competing hues.
