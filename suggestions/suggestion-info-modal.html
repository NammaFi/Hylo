Understanding Asset Card Metrics

Core Metrics
  • Asset Name: The yield-bearing token (Base Asset)
  • Maturity Timer: Time till yield token expires
Price & Range
  • Price Range: Expected price fluctuation range based on Implied Yield Range
Yield & Leverage
  • Underlying APY: Annual Percentage Yield at current market conditions, a 7-day Average
  • Implied Yield: Market's expected yield based on YT pricing
  • Leverage: How much your yield is amplified (e.g., 2x = double the base yield)
Performance Metrics
  • Expected Recovery Yield: Total Percentage Recovery of underlying asset possible through Yields, if hold till maturity
  • Daily Decay Rate: Daily percentage decrease in yield value due to time passing, for the same Implied Yield
  • Upside Potential: Maximum potential gain possible for today if implied yield increases to upper range
  • Downside Risk: Maximum potential loss possible for today if implied yield decreases to lower range
Points Tracking
  • Expected Points/Day: Projected reward points earned daily (scales with deposit amount)
  • Total Expected Points: Total points by maturity date (scales with deposit amount)
  • Boost: Additional multiplier for point earnings (if applicable)
Last Day Value
  • Current IY: Expected YT value based on current market yield
  • Lower IY: Expected YT value if yield drops to lower bound
  • Shows percentage of your investment remaining at 1 day from maturity
💡 Tip: All point calculations update based on your "Deposit Amount" setting
💡 Note: Data updates every 5 minutes. When someone visits and if last updated >10 minutes — hard refresh (1.5-2 Minutes)

What changed

  • Flat bg-surface background — matches cards & panels across all pages
  • Top gradient stripe (violet → teal) — same as tool cards & table groups
  • Subtle border (border-2) instead of colored purple border
  • Icon in a violet-tinted box — same pattern as tc-icon on Home page
  • Plain white title — no gradient text, matches dashboard-title style
  • Close button styled as a bordered pill — matches the small UI elements
  • Section labels use mono + line divider — same as source-group-header
  • Each metric sits in its own bg-surface-2 row — easier to scan, consistent with card interiors
  • Tip bar pinned at bottom with bg-surface-2 — clearly separated from content
  • All colors from design token system — no one-off rgba values
